# Phase C: Frontend & Asset Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Consolidate inline JavaScript into shared modules, tighten the CSS design system with dark mode and self-hosted fonts, and add an esbuild asset pipeline for minification.

**Architecture:** Three sequential sub-phases — C.1 extracts inline scripts into standalone JS files loaded on every page, C.2 improves CSS and self-hosts fonts, C.3 adds esbuild bundling. Each sub-phase is a commit checkpoint.

**Tech Stack:** Vanilla JS (ES5 compat), CSS custom properties, esbuild (via npx), WOFF2 font files

**Design spec:** `docs/superpowers/specs/2026-04-14-systematic-improvement-design.md` (Phase C section)

## Task 1: Extract Inline Scripts to Shared JS Modules

Create three shared JS files that replicate the behavior of the inline scripts currently generated by `render_filter_script()`, `render_table_enhancements_script()`, and `render_mobile_drawer_script()` in `scripts/builders/html_utils.py`.

The shared JS discovers configuration from DOM data attributes rather than being parametrised by Python. Pages that still use inline scripts (body_html pages) will have both — the shared JS adds listeners that produce the same behavior, which is safe.

**Files to create:**
- `site/assets/filters.js` — table filtering with shared state persistence
- `site/assets/tables.js` — sortable headers, filter chips, presets
- `site/assets/drawers.js` — mobile detail drawer

**Files to modify:**
- `scripts/templates/base.html` — add script references

READ `scripts/builders/html_utils.py` lines 382-623 to understand the three inline script generators. The shared JS must replicate their exact behavior but discover table IDs and configuration from the DOM.

### Implementation approach

- `filters.js`: discovers filterable tables by finding `[data-table]` controls, reads filter fields from row data attributes, reads shared filters from a `data-shared-filters` attribute on the table
- `tables.js`: discovers tables with `data-presets` attribute for preset config, applies sortable headers to all `.dense-table` elements
- `drawers.js`: discovers all `.dense-table` elements, reads column labels from `<thead>`, creates a single drawer element

Add to `scripts/templates/base.html` before `shell.js`:
```html
<script src="assets/filters.js"></script>
<script src="assets/tables.js"></script>
<script src="assets/drawers.js"></script>
```


## Task 2: Self-Host Fonts and Add Dark Mode

**Files to create:**
- `site/assets/fonts/source-sans-3-regular.woff2`
- `site/assets/fonts/source-sans-3-semibold.woff2`
- `site/assets/fonts/source-sans-3-bold.woff2`

**Files to modify:**
- `site/assets/styles.css` — self-hosted @font-face, dark mode
- `scripts/templates/base.html` — remove Google Fonts links

### Steps

1. Download WOFF2 font files for Source Sans 3 weights 400, 600, 700 from Google Fonts CDN to `site/assets/fonts/`
2. Replace the "Source Sans 3 Fallback" `@font-face` block in styles.css with proper self-hosted `@font-face` declarations pointing to the local WOFF2 files
3. Add dark mode via `@media (prefers-color-scheme: dark)` overriding CSS custom properties (`--bg`, `--surface`, `--ink`, `--accent`, etc.) with dark-appropriate values, plus dark badge colors and dark table striping
4. Remove the three Google Fonts `<link>` tags from `scripts/templates/base.html` (`preconnect` and CSS link)
5. Rebuild and verify

### Commit

```bash
git add site/assets/fonts/ site/assets/styles.css scripts/templates/base.html site/ scripts/tests/baseline_snapshot.json
git commit -m "feat: C.2 — self-host fonts, add dark mode, remove Google Fonts dependency"
```

## Task 3: Add esbuild Asset Pipeline

**Files to create:**
- `scripts/build_assets.sh` — esbuild bundler script

**Files to modify:**
- `.gitignore` — add `node_modules/` and `site/assets/dist/`

### Steps

1. Create `scripts/build_assets.sh` that uses `npx --yes esbuild` to:
   - Minify each JS file (shell.js, filters.js, tables.js, drawers.js) to `site/assets/dist/`
   - Minify styles.css to `site/assets/dist/`
   - Copy fonts directory to `site/assets/dist/fonts/`
2. Make it executable: `chmod +x scripts/build_assets.sh`
3. Add `node_modules/` and `site/assets/dist/` to `.gitignore`
4. Test: `bash scripts/build_assets.sh` — verify minified files are created
5. Verify site still works with all checks

Note: The unminified source files are what GitHub Pages serves. The dist/ directory is for optional production optimization. Templates continue to reference the unminified files.

### Commit

```bash
git add scripts/build_assets.sh .gitignore
git commit -m "feat: C.3 — add esbuild asset pipeline for JS/CSS minification"
git push origin main
```
